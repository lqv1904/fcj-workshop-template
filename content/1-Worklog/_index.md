**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Working with AWS Core Services: IAM, VPC, EC2, and RDS](1.2-week2/)

**Week 3:** [Implementing AWS Infrastructure Scaling, Cost Management, and Monitoring](1.3-week3/)

**Week 4:** [Configuring Hybrid DNS, AWS CLI Automation, and Automated Data Protection](1.4-week4/)

**Week 5:** [VM Migration, Docker Containerization, and Exploring ECS with CI/CD Pipelines](1.5-week5/)

**Week 6:** [Advanced Networking with VPC Peering and Transit Gateway, Security Monitoring, and Cost Optimization via AWS Lambda](1.6-week6/)

**Week 7:** [Deploying CI/CD Pipelines with AWS Developer Tools, Integrating Hybrid Storage, and Managing Amazon FSx](1.7-week7/)

**Week 8:** [Web Application Security with AWS WAF, Resource Management using Tags, and Monitoring with Grafana](1.8-week8/)

**Week 9:** [Advanced Access Control with IAM, System Automation with Systems Manager, EC2 Optimization, and Data Encryption with KMS](1.9-week9/)

**Week 10:** [AWS Cost Visualization, Building a Data Lake, and In-Depth Infrastructure Monitoring with CloudWatch](1.10-week10/)

**Week 11:** [System Integration, Functional Testing, and AWS Infrastructure Configuration](1.11-week11/)

**Week 12:** [Overall Evaluation, Documentation Finalization, and Final Project Reporting](1.12-week12/)
